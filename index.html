嵌入式比赛总结

May 10th, 2014 | Comments

劳动节已经过去，五一假期一直在实验室呆着，每天忙忙碌碌，服务器端的例程耗时两天总算弄懂了。大致的框架是
下端zigbee终端设备和各传感器之间通过串口采集传感器的数据，zigbee上跑的是zstack协议栈，关于zstack协议栈，以后再进行分析，这个协议栈总的来说是根据zigbee标准制定的用来做物联网的。当zigbee终端设备采集到数据之后，会通过无线网络传到协调器处，协调器实际也是一个zigbee设备，只是通过软件对其进行了类别划分，它可以管理其他的zigbee设备，它收到终端设备发过来的数据后会在通过串口传输到arm网关部分，在arm上可以对数据进行处理，包括数据的显示，分析，开关控制，报警等功能。
对于设备中arm上的开机跑的智能家居的例程，其逻辑主要如下：
arm上实现了两个程序，一个是c语言实现的服务器端，另一个是qt实现的客户端，qt并没有直接通过串口和zigbee或者rfid或者gprs或者蓝牙通信，一切的数据获取，报警短信，开关控制都要通过服务器端的程序。服务器端和zigbee，服务器端和qt客户端都有一个简单地协议，服务器和qt端通过socket通信。并且两方都通过select机制监听可读事件。服务器端使用了libev这个库，据说这个库为libevent的升级版。花了两天的时间大致了解了一下libevent的源代码，参考文章如下
libevent代码分析
其中越往下看，越发现数据结构与算法才是根本。涉及到了链表，堆排序等。看这种代码确实能够学到很多东西，借着这个机会熟悉了一下二叉树，以前并不知道堆是怎么存储的，现在才知道其实堆就是用二叉树这种数据结构来实现的（囧）,因此操作堆就是操作二叉树，并且堆总是一个完全二叉树， 对其可以进行遍历（前序，中序，后序等），还有查找，插入，删除等一系列操作。而二叉树的操作又是建立在迭代的基础之上。libevent库中使用了最小堆排序。对于不同的事件，IO，定时器，信号，都有不同的数据结构与算法来实现，至今也只是了解了一点皮毛。
服务器端的程序大量使用了多线程，这个时候我们就要小心线程之间的数据共享等问题。
比赛中发现自己真是没有写多少代码，总是在别人的基础上进行修改，增添一些自己的功能。倒是下端的zigbee对于其数据的传输逻辑了解了不少。包括串口的初始化，任务id的注册，怎么通过无线发送数据，从无线端传过来的数据怎么处理，不同的设备如何识别。终端设备如何加入自己的传感器以及与传感器之间的接口如何实现等现在了解的都差不多了。但是对于大量的节点身份如何识别，如何选择路径，何时加入网络，何时选择网络，传感器的自组网算法等都有待进一步的学习。最近也了解了一下contiki这个开源的操作系统，跑了个串口的例子，没想到一下就跑通了。所以打算深入的了解一下它的源代码，等以后有空了在作总结。
慢慢的来吧，虽说快要结束了，但是学习这件事情是急不得的。

#include <iostream>
using namespace std;
//参数为数组和其大小
void insert(int *data,int n)  
{
    int target;
    for(int i=1; i<n; i++) 
    {
        int j = i;
        target = data[i]; //将要排序的元素
        while(j > 0 && target < data[j-1])  //排序的条件，j大于0，并且将要排序的元素要找到从左边起第一个大于它的元素，放在这个元素的左边，
        {
            data[j] = data[j-1];
            j--;
            //后边的元素整体后移一个
        }
        data[j] = target;
    }
    for(int i = 0;i < n; i++)
        cout << data[i] <<",";
}

void main()
{
    int data[]={6,4,2,3,1,7,9};
    int n=7;
    cout << "排序前：";
    for(int i = 0;i < 7; i++)
        cout << data[i] <<  ",";

    cout << "排序后：";
    insert(data,n);
    cout << endl;
}
//平均时间复杂度O(n2)

int search(int arr[],int k,int n);
int main()
{

int arr[15]={0,1,2,2,2,2,3,3,3,4,5,6,7,8,9};
int a=search(arr,2,15);
if(a==-1)
    cout<<"出错 "<<endl;
else 
    cout<<"数组下标："<<a<<endl;
return 0;
}

int search(int arr[],int k,int n)
{
    int start,end,mid;
    start=0;
    end=n-1;
    while(start<=end)   //判断条件
    {
        mid=(start+end)/2;  //取中间值
        if(k==arr[mid])
        {
            while(k==arr[mid])
            {
                mid++;
            }
    /*
        加上while可以获取最右边的第一个出现的索引，不加while为最左边第一次出现的索引
    */
            return mid-1;
        }
        else if(k<arr[mid])  //继续往左查找
            end=mid-1;
        else
            start=mid+1;  //往右查找

    }
return -1;
}
